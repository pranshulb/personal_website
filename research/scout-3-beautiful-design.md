## 1. Derek Sivers — sive.rs

**URL:** https://sive.rs/

**Who:** Writer, musician, entrepreneur. Author of multiple books including "How to Live" and "Useful Not True". Former circus performer, CD Baby founder.

### Design/Layout
- **Radical minimalism.** Just text and links. No images (except small headshots in one section). No CSS framework bloat.
- Black text on white background. Blue underlined links. It's 1995 web but done *intentionally*.
- Simple semantic HTML. Fast as hell. No JavaScript needed for core functionality.
- Organized by sections: "Me in 10 seconds", books, articles, projects, interviews, photos.
- Content-first architecture — zero design ego, 100% legibility and accessibility.

### What Makes It Special
This is **anti-design as design philosophy**. Derek's site is a middle finger to modern web bloat while being incredibly functional and pleasant to use. It loads instantly, works on any device, and will outlive every JavaScript framework.

The genius is that it's **not lazy** — it's a deliberate aesthetic choice from someone who could afford fancy design but chooses simplicity. That intentionality makes it beautiful.

**The vibe:** "I have important things to say and I'm not going to let design get in the way."

### Tech Stack
- Plain HTML/CSS
- No visible framework
- Likely static site generator or very simple server-side rendering
- Minimal JavaScript (if any)
- Self-hosted (based on domain)

### Quote from the site
> "Everything here is written by me, personally, not an AI."

## 2. Bartosz Ciechanowski — ciechanow.ski

**URL:** https://ciechanow.ski/

**Who:** Technical educator. Creates interactive articles about science and engineering.

### Design/Layout
- **Interactive 3D demonstrations** embedded throughout long-form articles
- Clean, article-based layout with heavy whitespace
- Scroll-based narrative with interactive widgets that respond to user input
- Custom-built visualizations (gears, optics, GPS, mechanical systems)
- Typography-forward design with generous line height and readable measure

### What Makes It Special
This is **web design as pedagogy**. Each article contains dozens of custom-built interactive diagrams that you can drag, rotate, and manipulate. Want to understand how GPS works? He built a 3D solar system simulator. Want to learn about cameras? Interactive lens diagrams.

The interactive elements aren't gimmicks — they're **integral to understanding**. You literally can't explain these concepts as well without them.

It's the Bret Victor vision of "explorable explanations" executed flawlessly. Each article probably took weeks to build, combining deep technical knowledge with design craft.

**The vibe:** "What if textbooks were alive?"

### Tech Stack
- Custom JavaScript for all interactive elements
- WebGL for 3D visualizations (likely)
- Canvas API for 2D diagrams
- Custom physics simulations
- Probably vanilla JS or minimal framework — optimized for performance
- Static HTML with embedded interactives

### Notable Articles
- "Moon and Earth" — Full orbital mechanics simulation
- "Lights and Shadows" — Interactive 3D rendering
- "GPS" — Solar system and satellite visualization

## Common Threads Across Exceptional Personal Sites

### 1. **Intentionality**
Every design choice is deliberate. Whether minimal or complex, nothing is accidental.

### 2. **Content First**
The design serves the content and the person behind it. No template thinking.

### 3. **Performance**
Fast sites. No bloat. Respect for the user's time and bandwidth.

### 4. **Authenticity**
These sites feel like the people who made them. Derek's minimalism, Bartosz's interactivity, Frank's restraint — all genuine expressions.

### 5. **Longevity**
Built to last. Simple tech stacks that won't break in 6 months when the framework updates.

### 6. **Craft**
Whether it's typography, code, or interactive design, these sites show **mastery**. You can tell someone who knows what they're doing made this.

### 7. **Independence**
These are *personal* sites. Not portfolio templates. Not Medium blogs. Personal URLs, personal control.

---

## Design Principles to Steal

1. **Start with text.** If it doesn't work as plain HTML, adding design won't save it.

2. **Speed is a feature.** Sub-second load times earn trust.

3. **Whitespace is content.** Negative space does as much work as positive space.

4. **Typography is 95% of design.** Get this right and you're most of the way there.

5. **One unique element is enough.** Derek has radical simplicity. Bartosz has interactivity. TCI has the spiral. You don't need ten clever ideas.

6. **Build to evolve.** Your site will change. Design for growth, not perfection.

7. **Resist trends.** These sites will look good in 2030 because they're not chasing 2025 aesthetics.

---

## Tech Stack Philosophy

### What Works
- Static site generators (fast, simple, durable)
- Minimal JavaScript (progressive enhancement)
- Semantic HTML
- Hand-crafted CSS
- Standard web fonts or system fonts
- Self-hosting when possible

### What to Avoid
- Heavy JavaScript frameworks for content sites
- Template builders (Wix, Squarespace) if you can code
- Design trends (parallax, mega-animations, etc.)
- External dependencies you don't control
- Anything that slows you down
