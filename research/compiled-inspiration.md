## 🔑 KEY PATTERNS

**What makes a great personal site:**
- Workshops, not showrooms — messy, evolving, real
- Content-first — the design serves the person, not the other way around
- One unique element is enough — don't need ten clever ideas
- Build to evolve — not frozen perfection
- Speed is a feature — sub-second loads earn trust
- Strong opinions — not trying to appeal to everyone
- Mix of polish and raw — room for both essays and rough notes

**Tech stack patterns:**
- Static site generators dominate (Quartz, Astro, Jekyll, Next.js)
- Markdown-first content
- Minimal JavaScript unless interactive elements need it
- Self-hosted or GitHub Pages/Vercel
- Hand-crafted CSS > frameworks

**What to avoid:**
- "Hi, I'm a Full Stack Developer" energy
- Perfect portfolio cases with no personality
- SEO-optimized blog titles
- WordPress default themes
- Hiding all rough edges
